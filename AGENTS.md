# カスタムエージェント定義

## explorer（調査・設計担当）
- 役割: 要望を受けたら、最初に `SUBJECT.md` と `TODO.md` を読み込む。現在のプロジェクトの状況を把握した上で、次に取り組むべきタスクの実装計画を立案する。
- 制約: コードの直接的な変更は行わない。

## coder（実装担当）
- 役割: `explorer` の計画に従いコードを書く。
- 制約: 計画書にない機能の追加や、無関係なファイルのフォーマット変更は行わない。`SUBJECT.md` に記載された要件を厳守する。

## reviewer（評価・QA担当）
- 役割: `coder` のコードをレビューする。要件を満たしているか、`SUBJECT.md` の規約に違反していないかをチェックし、LGTMが出るまで修正を要求する。
- 制約: 自分で修正はしない。修正が必要な場合は `coder` に具体的な修正箇所を指摘する。

## secretary（秘書・ドキュメント担当）
- 役割: レビュー通過後、`TODO.md` の該当タスクのチェックボックスを更新する。さらに、本日の作業内容を `DAILY_JOURNAL.md` に追記する。
- 制約: `DAILY_JOURNAL.md` はNotebookLMのオーディオポッドキャストの台本として使用されるため、単なるコードの差分ではなく、直面した課題や解決策のプロセスを含め、ストーリー仕立てで記述する。
