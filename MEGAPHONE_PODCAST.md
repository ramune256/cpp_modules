# Megaphone を理解するためのポッドキャスト台本

## オープニング

こんにちは。今日は C++ Module 00 の最初の課題、Megaphone を題材に、C はよく知っているけれど C++ は初めて、という人向けに「何が C と同じで、何が C++ らしいのか」を整理していきます。

この課題でやること自体はとても小さいです。コマンドライン引数を受け取って、全部大文字にして出力する。ただし引数がひとつもなければ、`* LOUD AND UNBEARABLE FEEDBACK NOISE *` という決まった文字列を表示する。それだけです。

でも、この小ささがちょうどいい。C++ の入り口で覚えるべき考え方が、かなり凝縮されています。

## この課題のゴール

まず、課題の要求を C 的に言い換えてみます。

`main(int argc, char **argv)` を受け取る。  
`argc == 1` なら定型メッセージを出して終了する。  
そうでなければ `argv[1]` から `argv[argc - 1]` までを順番に走査し、各文字を大文字化してそのまま連結して出力する。

ここまでは、ほとんど C の問題に見えます。実際、制御構造も文字列の走査も発想は同じです。

では C++ の課題として何が違うのでしょうか。ポイントは、「C でできることを C っぽく書くな、C++ の流儀で書け」という点です。

## 実装の全体像

今回の実装は次のような形です。

```cpp
#include <cctype>
#include <iostream>

int main(int argc, char **argv)
{
	if (argc == 1)
	{
		std::cout << "* LOUD AND UNBEARABLE FEEDBACK NOISE *" << std::endl;
		return 0;
	}

	for (int i = 1; i < argc; ++i)
	{
		for (int j = 0; argv[i][j] != '\0'; ++j)
			std::cout << static_cast<char>(std::toupper(static_cast<unsigned char>(argv[i][j])));
	}
	std::cout << std::endl;
	return 0;
}
```

短いですね。では、この短いコードの中にある C++ の入口を順番に見ていきます。

## C と同じところ

最初に安心材料から確認します。

`argc` と `argv` の意味は C と同じです。  
`for` で配列を回す感覚も同じです。  
`argv[i][j] != '\0'` で文字列終端を見ているのも、完全に C と同じです。

つまりこの課題は、「C の知識が無駄になる」のではなく、「C の知識の上に C++ の書き方を重ねる」練習です。

## 最初の C++ らしさその1: `#include <iostream>`

C ならたぶん `#include <stdio.h>` を入れて `printf` を使いたくなります。でもこの課題では、そう書かないのが大事です。

C++ では標準出力に対して `std::cout` を使います。

```cpp
std::cout << "hello" << std::endl;
```

この `<<` はビットシフトではなく、ストリームにデータを流し込む演算子です。C の `printf("value=%d\n", x);` に比べると、型ごとの書式指定子を覚えなくていいのが特徴です。

C では「文字列フォーマットを書く」という意識が強いですが、C++ では「出力ストリームに値を順番に流す」という発想になります。

## C++ らしさその2: `std::cout` と `std::endl`

引数がないときの処理はこうです。

```cpp
std::cout << "* LOUD AND UNBEARABLE FEEDBACK NOISE *" << std::endl;
```

ここで `std::endl` は改行を入れるマニピュレータです。見た目は少し不思議ですが、役割としては「改行して一区切りつける」ものです。

C の `"\n"` と同じ場面で使えますが、C++ ではこういう「ストリームに特殊な動作を流し込む」書き方が出てきます。

## C++ らしさその3: `std::` という名前空間

このコードでは `cout` ではなく `std::cout`、`endl` ではなく `std::endl`、`toupper` も `std::toupper` と書いています。

これは C++ の名前空間です。標準ライブラリの名前は `std` という箱の中に入っている、と考えるとわかりやすいです。

この課題群では `using namespace std;` が禁止です。なので毎回 `std::` を書きます。最初は少し面倒ですが、「どの名前が標準ライブラリ由来なのか」が明確になるので、慣れる価値があります。

## C++ らしさその4: C の関数にも C++ 版の入口がある

文字の大文字化には `std::toupper` を使っています。

```cpp
std::toupper(...)
```

ここでインクルードしているのは `<ctype.h>` ではなく `<cctype>` です。これも C から C++ への移行でよく出るポイントです。

C の標準ヘッダに対応する C++ 版は、しばしば `<c...>` という形になります。

`<stdio.h>` ではなく `<cstdio>`  
`<ctype.h>` ではなく `<cctype>`

こういう形で「C の機能を C++ の文脈で使う」入口が用意されています。

## なぜ `static_cast<unsigned char>` が入っているのか

このコードでいちばん初心者が見落としやすいのがここです。

```cpp
static_cast<char>(std::toupper(static_cast<unsigned char>(argv[i][j])))
```

長いですね。でも意味はあります。

`toupper` 系の関数は、負の `char` が入ると未定義動作の原因になり得ます。C に慣れている人ほど「ASCII しか来ないから平気だろう」と流しがちですが、C++ でもこの注意点は同じです。

そこでまず `argv[i][j]` を `unsigned char` に変換してから `std::toupper` に渡します。戻り値は `int` なので、出力用に最後 `char` に戻しています。

ここでの学びは二つあります。

ひとつは、C++ でも文字処理の落とし穴は残るということ。  
もうひとつは、C++ ではキャストを明示的に、意図がわかる形で書く文化が強いということです。

## `static_cast` は C のキャストとどう違うのか

C ならこう書きたくなるかもしれません。

```cpp
(char)some_value
```

でも C++ では、今回のような通常の型変換には `static_cast<Type>(value)` を使うことが推奨されます。

理由は単純で、何をしているかが読みやすいからです。  
「強引に型を変えた」のではなく、「静的にわかる範囲の変換をしている」と伝わります。

この課題で `static_cast` が使われているのは、C++ の作法に少しずつ慣れるためにも良い選択です。

## ロジック自体はとても素直

大文字化の本体は二重ループです。

外側のループで引数を順番に処理する。  
内側のループで各引数の文字列を 1 文字ずつ処理する。

つまり考え方はこうです。

1. 最初の引数 `argv[1]` から始める
2. 文字列の終端 `'\0'` まで 1 文字ずつ見る
3. 各文字を大文字化して即座に出力する
4. 次の引数へ進む
5. 最後に改行して終わる

C の配列走査に慣れていれば、ロジックの理解自体で詰まることはほぼありません。

## あえて `std::string` を使っていない理由

C++ を始めたなら、すぐ `std::string` を使いたくなる人もいると思います。実際、将来的には文字列操作に `std::string` を使う場面は多いです。

ただ、この課題の段階で `argv` を直接なめる実装には意味があります。C の文字列と C++ の標準出力の橋渡しになるからです。

つまり、

入力の受け方は C に近い。  
出力とライブラリの使い方は C++ に寄せる。

この混ざり方が、Module 00 の入口としてちょうどいいわけです。

## Makefile から見えること

この課題では `Makefile` も提出物です。設定はこうです。

```make
NAME = megaphone
CXX = c++
CXXFLAGS = -Wall -Wextra -Werror -std=c++98
```

ここで重要なのは `-std=c++98` です。現代の C++ ではなく、かなり初期の規格で書けることが求められています。

つまり、「新しい便利機能を覚える」より先に、「C++ の基本的な文法と標準ライブラリの入口に慣れる」ことが目的です。

この制約のおかげで、逆に学ぶ焦点がぶれません。

## C の人が最初に意識すると良い変化

この課題から読み取れる、C から C++ への最初の変化をまとめるとこうなります。

1. 出力は `printf` ではなく `std::cout`
2. 標準ライブラリの名前には `std::` を付ける
3. C のヘッダより C++ 版ヘッダを優先する
4. キャストは C 風ではなく `static_cast` を使う
5. 同じ処理でも「C++ らしい書き方」がある

ロジックよりも、書き方の姿勢を学ぶ課題だと言ってもいいです。

## この課題の先に何があるのか

Megaphone 自体は小さなプログラムですが、ここで触れた要素はこの先ずっと出てきます。

`std::cout` は今後も頻繁に使います。  
`std::` を付ける癖もずっと必要です。  
「C の知識を持ち込みつつ、C++ の流儀で書く」という感覚も、以後のクラス実装で効いてきます。

つまりこの課題は、ただの文字変換問題ではなく、「C++ に入るときの姿勢を切り替えるスイッチ」なんです。

## クロージング

今日は Megaphone を通して、C に慣れた人が C++ の最初の一歩で押さえるべきポイントを見てきました。

やっていることは単純でも、`std::cout`、`std::endl`、`std::toupper`、`<cctype>`、`static_cast`、`std::` という名前空間。こうした要素が一気に登場するので、実はかなり密度の高い入門課題です。

もしこの課題を自分で読み直すなら、「どうやって大文字化しているか」だけでなく、「なぜ C の書き方ではなく、この C++ の書き方を選んでいるのか」を意識してみてください。そこがわかると、Megaphone は単なる小課題ではなく、C++ の入口としてかなりよくできた教材に見えてきます。

それでは今回はここまでです。
